## 1. プロジェクト概要

### 1.1 背景

東京都の駅情報・バス路線・鉄道路線の関係を横断的に調べられる手段がない。本アプリはGraphQLサーバーから3種類のデータ（駅・バス・路線）をまとめて取得し、それらの関係を視覚的に表示する情報閲覧アプリである。

### 1.2 目的

- 東京都の駅を路線別に一覧表示し、路線名ヘッダーで視覚的に路線の切れ目を示し、目的の路線へ素早くアクセスできる
- 駅に接続するバス路線の終点駅を一目で確認できる
- バス終点駅にどの鉄道路線が乗り入れているかをダイアログで確認し、その路線へジャンプできる

### 1.3 対象ユーザー

- 開発者自身

### 1.4 スコープ（対象範囲）

**対象範囲（作るもの）:**

- 東京都の駅・バス・路線データの取得と一覧表示
- 路線別グループ化と路線ナビゲーター
- バス接続情報の展開表示
- バス終点駅の電車情報ダイアログ
- ダイアログからの路線スクロールジャンプ
- Pull to Refresh
- エラー時のリトライ表示
- ダークテーマ

**対象外（作らないもの）:**

- 東京都以外の路線・駅への対応
- 地図表示
- 時刻表の表示
- お気に入り・履歴機能
- ルート検索機能
- オフライン対応（ローカルキャッシュは graphql_flutter の HiveStore に任せる）

## 4. 機能要件

### F-01 データ取得

```
概要:    アプリ起動時にGraphQLサーバーへクエリを送り、
         駅・バス・路線の3種類のデータを1回のリクエストで取得する
入力:    なし（自動取得）
処理:    GraphQL クエリを送信し、レスポンスを受け取る
出力:    stations・buses・trains の3種類のリスト
```

**取得するデータ:**

| データ種別 | 主要フィールド |
|---|---|
| stations | id, stationName, prefecture, trainNumber |
| buses | id, endA, endB |
| trains | trainNumber, trainName |

**異常系:**
- 通信エラー時はエラーメッセージとリトライボタンを表示する
- データ取得中はローディングインジケーターを表示する

### F-02 路線別駅一覧表示

```
概要:   取得した駅を路線ごとにグループ化し、各グループの先頭に路線名ヘッダーを挿入して縦リストに表示する
入力:   F-01で取得した stations と trains
処理:   1. trainNumber ごとに駅をグループ化する
        2. trains の並び順に従って路線を並べる
        3. trains に存在しない trainNumber は末尾に追加する
        4. 各路線グループの先頭に路線名ヘッダー行を挿入する
        5. trainNumber が空の駅は最後に並べる（ヘッダーなし）
出力:   路線名ヘッダーと駅が交互に並ぶ混合リスト
```

**表示ルール:**

| アイテム種別 | 表示ウィジェット | 内容 |
|---|---|---|
| 路線名ヘッダー | Container | 電車アイコン + 路線名（太字）+ 路線番号。各路線グループの先頭に表示 |
| バス接続あり | Card + ExpansionTile | バスアイコン + 駅名 + 路線名 + バス件数。タップで展開 |
| バス接続なし | ListTile | 駅名 + 路線名 |

### F-03 路線ナビゲーター表示

```
概要:   画面上部に路線アイコンを横スクロールリストで表示する
入力:   trainNumbersForHeader（路線番号リスト）、trainMap（路線番号→路線名）
処理:   各路線の trainName を CircleAvatar に表示する
        CircleAvatar の下に trainNumber をラベルとして表示する
出力:   横スクロール可能な路線アイコン一覧（高さ100）
```

### F-04 路線ジャンプ

```
概要:   路線アイコンをタップするとその路線の路線名ヘッダー行へスクロールする
入力:   タップされた路線の trainNumber
処理:   firstIndexByTrainNumber[tn] でリスト内のヘッダー行インデックスを取得する
        ItemScrollController.scrollTo() でそのインデックスへアニメーション付きでスクロールする
出力:   リストがその路線の路線名ヘッダー行の位置にスクロールされる
```

**アニメーション設定:**

| 設定項目 | 値 |
|---|---|
| duration | 450ms |
| curve | Curves.easeInOut |
| alignment | 0.0（先頭に揃える） |

### F-05 バス接続展開

```
概要:   バス路線を持つ駅をタップすると、接続するバスの終点駅一覧を展開表示する
入力:   busMap（駅名→終点駅リスト）、駅名
処理:   busMap[stationName] で終点駅リストを取得する
        ExpansionTile の children にリスト表示する
出力:   終点駅ごとに ListTile（終点駅名 + 情報ボタン）を表示する
```

**動作ルール:**
- busMap は双方向に構築する（endA→endB、endB→endA の両方を登録）
- バス接続がない駅は展開不可（ExpansionTile ではなく ListTile を表示）
- バス件数は subtitle に `N 件` と表示する

### F-06 電車情報ダイアログ

```
概要:   バス終点駅の情報ボタンをタップすると、
        その駅を通る電車の一覧をダイアログで表示する
入力:   終点駅名（endB）、stationTrainMap（駅名→路線番号リスト）、trainMap（路線番号→路線名）
処理:   stationTrainMap[endB] で路線番号リストを取得する
        各路線番号を trainMap で路線名に変換する
        AlertDialog に ListTile 形式で表示する
出力:   AlertDialog（タイトル: 「{駅名} の電車」、内容: 電車一覧）
```

**ダイアログの内容:**

| 要素 | 内容 |
|---|---|
| タイトル | `{終点駅名} の電車` |
| リストアイテム | 電車アイコン + 路線名（title）+ 路線番号（subtitle） |
| 電車情報なしの場合 | 「電車情報がありません」と表示 |
| ボタン | 「閉じる」ボタン（F-07 とは別にダイアログのみ閉じる） |

### F-07 ダイアログからジャンプ

```
概要:   ダイアログ内の電車をタップするとダイアログを閉じ、
        その電車の先頭駅の位置へスクロールする
入力:   タップされた電車の trainNumber、firstIndexByTrainNumber
処理:   Navigator.pop(ctx) でダイアログを閉じる
        firstIndexByTrainNumber[tn] で先頭インデックスを取得する
        _jumpToIndex() でスクロールする
出力:   ダイアログが閉じ、駅リストがその電車の先頭駅にスクロールされる
```

**動作ルール:**
- firstIndexByTrainNumber に対象の trainNumber が存在しない場合はスクロールをスキップする

### F-08 Pull to Refresh

```
概要:   画面を下に引っ張るとデータを再取得する
入力:   ユーザーの下方向スワイプ操作
処理:   refetch() を呼び出す
出力:   最新データで画面が更新される
```

### F-09 エラー表示とリトライ

```
概要:   APIエラー時にエラーメッセージとリトライボタンを表示する
入力:   GraphQL の result.hasException
処理:   エラーメッセージ・リトライボタン（再試行）を表示する
出力:   ユーザーがリトライボタンをタップすると refetch() が呼ばれる
```

### F-10 ダークテーマ

```
概要:   アプリ全体を黒ベースのダークテーマで表示する
設定:   themeMode: ThemeMode.dark（システム設定に関わらず常にダークモード）
```

## 6. データ設計

### 6.1 データの種類

| データ名 | 概要 | 主な項目 |
|---|---|---|
| 駅情報（stations） | 東京都の鉄道駅の基本情報 | ID、駅名、都道府県、路線番号 |
| バス情報（buses） | バス路線の両端の駅名 | ID、endA（一端）、endB（他端） |
| 路線情報（trains） | 鉄道路線の基本情報 | 路線番号、路線名 |

### 6.2 データの関係

```
┌──────────────────┐       ┌──────────────────┐
│    路線情報        │       │    駅情報          │
│                  │ 1   N │                  │
│  - 路線番号       │───────│  - 駅ID           │
│  - 路線名        │       │  - 駅名           │
│                  │       │  - 都道府県        │
└──────────────────┘       │  - 路線番号        │
                           └────────┬─────────┘
                                    │ 駅名で紐付け
                           ┌────────┴─────────┐
                           │   バス情報          │
                           │                  │
                           │  - endA（始点）   │
                           │  - endB（終点）   │
                           └──────────────────┘
```

### 6.3 アプリ内で計算するマップ一覧

> **補足：** APIから取得したデータをそのまま使うのではなく、画面表示や検索を効率化するために各種マップを事前に計算する。

| マップ名 | キー | 値 | 用途 |
|---|---|---|---|
| `busMap` | 駅名 | バス終点駅名のリスト | 駅に接続するバス終点を取得 |
| `trainMap` | 路線番号 | 路線名 | 路線番号から路線名を取得 |
| `stationTrainMap` | 駅名 | 路線番号のリスト | 駅を通る電車一覧を取得（ダイアログ用） |
| `stationsByTrain` | 路線番号 | 駅マップのリスト | 路線ごとの駅グループ |
| `trainNumbersForHeader` | （リスト） | 路線番号 | 上部ナビゲーターの表示順 |
| `firstIndexByTrainNumber` | 路線番号 | items リスト内のヘッダー行インデックス | スクロールジャンプ用（ヘッダー行を指す） |

### 6.4 busMap の双方向構築

> **補足：** バスデータの `endA`・`endB` は「路線の両端」を表すため、どちらの駅からでも相手側の駅を検索できるように双方向でマップを構築する。

```
buses: [{endA: "渋谷", endB: "中野"}, ...]

↓ 双方向でbusMapに登録

busMap["渋谷"] = ["中野"]
busMap["中野"] = ["渋谷"]
```

## 7. 外部インターフェース設計

### 7.1 GraphQL クエリ

**エンドポイント:** `http://49.212.175.205:8081/graphql`

**クエリ:**

```graphql
query {
  stations(prefecture: "東京都") {
    id
    stationName
    prefecture
    trainNumber
  }
  buses {
    id
    endA
    endB
  }
  trains {
    trainNumber
    trainName
  }
}
```

### 7.2 レスポンスの概要

| フィールド | 型 | 概要 |
|---|---|---|
| `stations` | 配列 | 東京都の全駅情報 |
| `buses` | 配列 | 全バス路線情報 |
| `trains` | 配列 | 全鉄道路線情報 |

### 7.3 通信シーケンス

```
Flutterアプリ              GraphQLサーバー
    │                          │
    │  POST /graphql            │
    │  （クエリ送信）            │
    │──────────────────────────►│
    │                          │
    │  JSON（stations/buses/trains）
    │◄──────────────────────────│
    │                          │
    │ データ加工・マップ構築      │
    │ 画面を描画                 │
```

## 8. 非機能要件

### 8.1 性能要件

| 項目 | 要件 | 備考 |

### 8.2 可用性要件

### 8.3 ユーザビリティ要件

| 項目 | 要件 |
|---|---|
| ナビゲーション | 路線アイコンのタップ1回で目的の路線の路線名ヘッダーへジャンプできること |
| 情報確認 | バス終点駅の電車情報が1タップで確認できること |
| テーマ | ダークテーマ（黒ベース）を採用すること |

## 9. 用語集

| 用語 | 意味 |
|---|---|
| GraphQL | 必要なフィールドを指定してデータを取得できるAPIのクエリ言語 |
| HiveStore | graphql_flutter が使うローカルキャッシュ。アプリ起動時に初期化が必要 |
| ExpansionTile | タップで開閉する折りたたみウィジェット |
| CircleAvatar | 円形にコンテンツを表示するウィジェット。本アプリでは路線アイコンとして使用 |
| endA / endB | バス路線の両端の駅名 |
| trainNumber | 鉄道路線を識別する番号 |
| firstIndexByTrainNumber | 路線番号 → 駅リスト内の先頭インデックスのマップ |
| Pull to Refresh | 画面を下に引っ張るデータ再取得操作 |
| FetchPolicy.networkOnly | キャッシュを使わず毎回ネットワークからデータを取得するポリシー |
